# @youneed/dom-provider-a11y ‚Äî Accessibility Reference

Accessibility helpers for `@youneed/dom` components, exposed as a single
namespaced object ‚Äî **`this.a11y`**. Source of truth: `packages/dom-provider-a11y/src/index.ts`,
`packages/dom-provider-a11y/README.md`. See [./dom.md](./dom.md) for the component model.

## Providers mechanism ‚Äî how `this.a11y` is attached

Same slot as every other youneed extension: `Component(tag, { providers: [...] })`.
A `ComponentProvider` has an `install(host)` hook that runs in the constructor and
`Object.defineProperty`s its contribution onto the host. Providers are orthogonal ‚Äî
compose `a11yProvider()` alongside `i18nProvider(...)`, etc., in one array.

```ts
import { Component, html, css } from "@youneed/dom";
import { a11yProvider } from "@youneed/dom-provider-a11y";

class Dialog extends Component("x-dialog", { providers: [a11yProvider()] }) {
  static styles = css`
    .panel { transition: transform .2s; color: #222; }
    @media (prefers-reduced-motion: reduce) { .panel { transition: none } }
    @media (prefers-color-scheme: dark)     { .panel { color: #eee } }
  `;
  onMount() {
    this.a11y.announce("Dialog opened");          // spoken by screen readers
    this.onCleanup(this.a11y.trapFocus());        // Tab stays inside; focus restored
  }
  render() {
    return html`<div class="panel" role="dialog" aria-modal="true" aria-label="Settings">
      <button>OK</button><button>Cancel</button>
    </div>`;
  }
}
```

## `this.a11y` ‚Äî the API

```ts
this.a11y.announce(msg, "polite" | "assertive"); // shared ARIA live region ("polite" default)
this.a11y.trapFocus();        // trap Tab; returns release() that restores prior focus (auto on disconnect)
this.a11y.releaseFocus();     // release the current trap (no-op if none)
this.a11y.focusFirst();       // focus first focusable descendant; false if none
this.a11y.setTabIndex(0, el); // tabindex on el (default the host)
this.a11y.makeFocusable(el);    // tabindex=0
this.a11y.makeUnfocusable(el);  // tabindex=-1 (still focusable programmatically)
this.a11y.roving("button", { orientation, loop, initial }); // ‚Üí RovingController
this.a11y.prefersReducedMotion; // reactive boolean; re-renders on OS change
```

Standalone (outside a component): `announce(msg, politeness?)`, `clearAnnouncer()`,
`onAnnounce(listener)`, `prefersReducedMotion()`.

## ARIA in shadow DOM

Live regions are appended **to `<body>`** (one hidden region per politeness,
reused) ‚Äî they sit in the light DOM precisely so screen readers see them despite
shadow encapsulation. Put `role` / `aria-*` attributes directly in your `html`
template; they live inside the shadow root but apply to the element they're on.
`aria-label` / `aria-modal` / `role="dialog"` as above. Note ARIA *references*
(`aria-labelledby`, `aria-controls`) cannot cross the shadow boundary by ID ‚Äî use
inline `aria-label` or light-DOM (`shadow: false`) elements when you need a cross-tree link.

## Focus management

```ts
// Modal dialog: trap on open, restore on close.
onMount()   { this.onCleanup(this.a11y.trapFocus()); } // focusFirst() runs, Tab cycles at edges
// trapFocus queries the render root (shadowRoot ?? host) for focusables and
// cycles Tab / Shift+Tab; the returned release() restores document.activeElement.
```

Roving tabindex makes a composite widget keyboard-operable ‚Äî only the active item
is in the Tab order (`tabindex=0`), the rest are `-1`, arrows / Home / End move focus:

```ts
class Toolbar extends Component("x-toolbar", { providers: [a11yProvider()] }) {
  onMount() {
    const ctl = this.a11y.roving("button", { orientation: "horizontal" }); // ‚Üê/‚Üí
    // ctl.activeIndex, ctl.setActive(i), ctl.destroy() ‚Äî auto-destroyed on disconnect
  }
  render() {
    return html`<button>Bold</button><button>Italic</button><button>Underline</button>`;
  }
}
```

`orientation`: `"horizontal"` (‚Üê/‚Üí), `"vertical"` (‚Üë/‚Üì), `"both"` (default).
`loop` (default `true`) wraps at the ends. `initial` (default `0`) is the first tabbable item.

## prefers-reduced-motion

`a11yProvider()` reflects `data-reduced-motion="true|false"` on the host and
re-renders on OS change (opt out with `a11yProvider({ reducedMotion: false })`).
Style off it, or read `this.a11y.prefersReducedMotion` in `render()`:

```ts
static styles = css`
  :host([data-reduced-motion="false"]) .spin { animation: spin 1s linear infinite }
`;
```

## CSS audit (dev-time)

Opt in with `a11yProvider({ audit: true })`. A microtask after mount it scans the
component's scoped sheets and `console.warn`s when they:

- **animate / transition** but ship no `@media (prefers-reduced-motion: reduce)` variant; or
- set **explicit colors** but aren't `color-scheme`-aware (no `color-scheme` decl, no
  `@media (prefers-color-scheme: ‚Ä¶)` rule). Colors via `var(--token)` / `transparent`
  / `currentColor` / `inherit` are treated as adaptive and never flagged.

```ts
a11yProvider({ audit: true });                       // both checks, console.warn
a11yProvider({ audit: { reducedMotion: false } });   // colorScheme check only
a11yProvider({ audit: { warn: msg => log(msg) } });  // redirect findings
```

Pure form for tests / build steps: `auditStyleSheets(sheets, { reducedMotion?, colorScheme?, label? })`
returns `A11yAuditFinding[]` (`{ kind: "reduced-motion" | "color-scheme", message, docs }`).

## Devtools ‚Äî `@youneed/dom-provider-a11y/devtools`

Capture/display split. `a11yPlugin()` is a `DevtoolsPlugin` (records every
`announce()` call); `a11yPanel()` is a `DevtoolsPanel` that tails announcements and
runs the CSS audit over **every** mounted component:

```ts
import { installDevtools, mountDevtoolsPanel, defaultPanels } from "@youneed/devtools";
import { a11yPlugin, a11yPanel } from "@youneed/dom-provider-a11y/devtools";
installDevtools({ plugins: [a11yPlugin()] });
mountDevtoolsPanel(document.body, { panels: [...defaultPanels(), a11yPanel()] });
```

Captured data also exposed directly: `a11yAnnouncements()`, `onA11yAnnouncements(fn)`,
`clearA11yAnnouncements()`. `@youneed/devtools` is an optional peer dependency.

## Answering style

`this.a11y.*` is the provider's surface ‚Äî never present its members as native
`HTMLElement`/`Component` methods. When a component animates or sets colors, point
to `audit: true`; for dialogs use `trapFocus()` + `announce()`; for composite
widgets (toolbars, menus, tabs) use `roving()`. Reach for inline `aria-label` over
ID references in shadow DOM.
