# @youneed/i18n — Internationalization Reference

A tiny, fully-typed translation core (`@youneed/i18n`) plus a DOM binding
(`@youneed/dom-provider-i18n`) that uses it inside `html` templates with reactive
re-render on locale change. Source of truth: `packages/i18n/src/index.ts`,
`packages/dom-provider-i18n/src/index.ts` (+ their READMEs). See [./dom.md](./dom.md)
for the component / providers model.

## Translation core — `createI18n`

Build a translator from a plain `resources` object. The keys you pass are
**type-checked and autocompleted** — inferred as a union of dotted leaf paths, so a
typo or a key present in only one locale is a compile error. **Params are inferred
per key** from the template's `{placeholders}`.

```ts
import { createI18n } from "@youneed/i18n";

const i18n = createI18n({
  resources: {
    en: { greeting: "Hello {name}", nav: { home: "Home" } },
    de: { greeting: "Hallo {name}", nav: { home: "Startseite" } },
  },
  locale: "en",
  fallbackLocale: "en",   // consulted when a key is missing in the active locale
});

i18n("greeting", { name: "Ada" }); // "Hello Ada"  ← autocompletes "greeting" | "nav.home"
i18n.setLocale("de");
i18n("nav.home");                   // "Startseite"
```

The instance is **both callable** (`i18n(key, params)`) **and an object**:

```ts
i18n.t(key, params?)  // same as calling it
i18n.locale           // active locale
i18n.locales          // every locale in resources
i18n.has(key)         // resolves in active locale or fallback (narrows the type)
i18n.setLocale(loc)   // switch + notify subscribers (no-op for an unknown locale)
i18n.subscribe(fn)    // run fn on every locale change; returns an unsubscribe
i18n.id               // stable per-instance id (correlates devtools events)
```

`missing: (key, locale) => string` (option) customizes the miss handler (default
returns the key). Interpolation: `{name}` slots filled from `params`; an absent slot
is left verbatim.

## Plurals

A message can be a `PluralForms` leaf (`{ one, few, many, other, … }`) instead of a
string. Pass `count` and the form is chosen via `Intl.PluralRules` for the active
locale — every CLDR category works with zero shipped data; only `other` is required.

```ts
const i18n = createI18n({
  resources: {
    en: { items: { one: "{count} item", other: "{count} items" } },
    ru: { items: { one: "{count} товар", few: "{count} товара", many: "{count} товаров", other: "{count} товара" } },
  },
  locale: "en",
});

i18n("items", { count: 1 }); // "1 item"
i18n("items", { count: 5 }); // "5 items"
i18n.setLocale("ru");
i18n("items", { count: 3 }); // "3 товара"  (few)
i18n("place", { count: 2, ordinal: true }); // ordinal rules with `ordinal: true`
```

`items` stays one autocompleted key (the form object is a leaf). Without `count` →
`other`. `isPluralForms(v)` is exported for runtime checks.

## DOM binding — `@youneed/dom-provider-i18n`

`i18n('key')` inside an `html` template returns a plain string, so it drops into any
hole. Reactivity is opt-in: re-render on every `setLocale(...)`.

### Recommended: `i18nProvider` (typed `this.i18n` + auto reactivity)

Plugs into the `Component(tag, { providers: [...] })` slot — the DOM analogue of a
server `Controller`'s `guards`/`interceptors`. Gives a **typed** `this.i18n` (keys
autocomplete) AND auto re-render on locale change; composes with other providers.

```ts
import { Component, html } from "@youneed/dom";
import { createI18n } from "@youneed/i18n";
import { i18nProvider } from "@youneed/dom-provider-i18n";

const appI18n = createI18n({
  resources: { en: { hello: "Hello {name}" }, de: { hello: "Hallo {name}" } },
  locale: "en",
});

class Greeting extends Component("x-greeting", { providers: [i18nProvider(appI18n)] }) {
  render() {
    return html`
      <p>${this.i18n("hello", { name: "Ada" })}</p>  <!-- ^ keys autocomplete + type-check -->
      <button @click=${() => this.i18n.setLocale("de")}>Deutsch</button>`;
    // clicking re-renders this component with the new locale (provider subscribed it)
  }
}
```

### App-wide global + `localized`

`provideI18n(instance)` installs an app-wide translator; the loose, **untyped**
`i18n(key, params?)` / `t(...)` read from it (convenient, no key autocomplete).
`localized(host)` subscribes the host and `requestUpdate()`s on locale change,
auto-unsubscribing on disconnect.

```ts
import { provideI18n, i18n, localized } from "@youneed/dom-provider-i18n";
provideI18n(appI18n);

class Greeting extends Component() {
  constructor() { super(); localized(this); }   // re-render on every setLocale
  render() { return html`<div>${i18n("hello", { name: "Ada" })}</div>`; }
}
```

For autocomplete with the global pattern, call your **typed** instance directly in
the template (`appI18n("hel…")`) and use `localized(this, appI18n)` for reactivity.

### Mixin form — `withI18n`

Same contribution as `i18nProvider`, as a base-class mixin (chainable with others):

```ts
import { withI18n } from "@youneed/dom-provider-i18n";
class Greeting extends withI18n(Component("x-greeting"), appI18n) {
  render() { return html`<div>${this.i18n("hello", { name: "Ada" })}</div>`; }
}
```

| API | meaning |
| --- | --- |
| `provideI18n(instance)` | install app-wide translator; returns it |
| `getI18n()` | active translator (throws if none provided) |
| `i18n(key, params?)` / `t(...)` | translate via app-wide translator (untyped) |
| `localized(host, instance?)` | re-render host on locale change; auto-cleanup |
| `i18nProvider(translator)` | `ComponentProvider` → typed `this.i18n` + reactivity |
| `withI18n(Base, translator)` | mixin form of the same |

## Devtools — `@youneed/dom-provider-i18n/devtools`

`i18nPlugin()` (capture, records every `t()` call framework-wide) +
`i18nPanel(i18n, { resources? })` (display: locale switcher, key browser with
per-locale gaps, call tail). Captured data also via `i18nUsage()`, `onI18nUsage(fn)`,
`clearI18nUsage()`.

## Companion packages

- `@youneed/logger-plugin-i18n` — translate log message keys.
- `@youneed/server-middleware-accept-language` — negotiate locale from `Accept-Language`.
- `@youneed/test-plugin-i18n` — assert translation parity in `@youneed/test`.

## Answering style

Prefer `i18nProvider(typedInstance)` — it's the one path that gives both key
autocomplete and reactivity with no boilerplate. The global `i18n(...)` is the
untyped convenience binding; reach for it only with `localized(this)`. Plurals are
`Intl.PluralRules`-driven (no data ships) — only `other` is required per locale.
