packages. One paradigm.

youneed is a TypeScript-first toolkit built on what the browser and Node already ship — Custom Elements, Shadow DOM, the HTTP server, the Speculation Rules API. No virtual DOM. Minimal runtime.

~170 packages in one monorepo · counted, not rounded up

Every package speaks the same shape. One paradigm: a factory returns a base class you extend, TC39 standard decorators register the members, a fluent builder wires it up. Component(tag), Controller(path), Page(url), Test() — the same three moves at every layer of the stack. What follows is one afternoon with it, in four numbered stages.

1.0 Define a component @youneed/dom · zero dependencies

Start in the browser. Component("x-counter") returns a base class; standard decorators register props and events; the template is a tagged literal. What renders is a real Custom Element with Shadow DOM — no virtual DOM in between.

// counter.ts
@Component.define()
class Counter extends Component("x-counter") {
  @Component.prop() count = 0;
  @Component.event() inc() { this.count++; }
  render() {
    return html`<button @click=${this.inc}>${this.count}</button>`;
  }
}

2.0 Add a typed server @youneed/server

Same shape, new layer. Controller("/counters") is a base class; decorated methods become routes; the context is typed end to end. The server runs on Node's own HTTP primitives.

// counters.ts
@Controller.define()
class Counters extends Controller("/counters") {
  @Controller.get("/:id")
  read(ctx: Context) {
    return { id: ctx.params.id, count: 0 };
  }
}

3.0 Render it server-side @youneed/ssr · builds on dom + server

A Page(url) renders the same component to Declarative Shadow DOM on the server, and ships a Speculation Rules hint so the next navigation is already prerendered when the reader clicks.

// home.ts
@Page.define()
class Home extends Page("/") {
  @Page.speculation({ prerender: "moderate" })
  render() {
    return html`<x-counter count="3"></x-counter>`;
  }
}

4.0 Test in the same paradigm @youneed/test

No fourth mental model. A suite extends Test(), decorated methods are the cases, and the component you defined at stage 1.0 mounts straight into the run.

// counter.spec.ts
@Test.define()
class CounterSpec extends Test("x-counter") {
  @Test.it("increments on click")
  async increments(ctx: TestContext) {
    const el = await ctx.mount(Counter);
    el.inc();
    expect(el.count).toBe(1);
  }
}

The dependency graph. ~170 packages in one monorepo, layered so you only pay for what you import. dom stands alone with zero dependencies; ssr builds on dom and server; devtools builds on dom and ssr.

The HTTP server holds around 50k requests per second on this repo’s local autocannon bench — a number from our machine, not a promise about yours. The bench script ships in the repo; run it and disagree.

One install gets you stage 1.0. The rest of the graph waits until you import it.
