The browser already shipped your framework.

Wrap. Don’t replace.

Components? Custom Elements are in every engine.

Scoped styles? Shadow DOM does it without a compiler.

Server rendering? Declarative Shadow DOM streams markup the parser understands — before any bundle lands.

Prefetching? Speculation Rules are an API, not a plugin.

youneed wraps these primitives instead of re-inventing them. No virtual DOM. Minimal runtime. TypeScript first.

And one paradigm everywhere: a factory returns a base class, TC39 standard decorators mark the parts, a fluent builder wires the rest. Component(tag), Controller(path), Page(url), Test() — they all feel the same.

— end of 01 · the argument

@Component.define()
class Counter extends Component("x-counter") {
  @Component.prop() count = 0;
  @Component.event() inc() { this.count++; }
  render() {
    return html`<button @click=${this.inc}>${this.count}</button>`;
  }
}

This is the whole component. A factory returns the base class, decorators declare the reactive surface, and the template compiles against the real DOM — there is no virtual one to diff.

The same shape runs the backend: Controller(path) classes on the built-in HTTP server hold ~50k req/s on the repo’s local autocannon bench.
